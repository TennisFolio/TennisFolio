새로 저장하거나 기존 이름 확인을 통과한 뒤 보이는 상세 화면입니다. 정보 저장과 참석 상태 선택은 한 블록에 모으고, `/manage`와 같은 현황을 보여주되 경기 생성 같은 owner 액션은 제외합니다.

운영 관리 버튼 배치 비교

`/meetings/{publicId}/manage`의 운영 관리는 5개 액션을 같은 크기로 쌓기보다, 실제 사용 빈도와 위험도를 기준으로 나누는 편이 깔끔합니다. 경기표 액션은 주요 행동, 공유/수정은 보조 도구, 마감은 상태 제어, 삭제는 위험 행동으로 분리합니다.

3. 더보기 메뉴

가장 간결하지만 숨겨진 액션이 생깁니다. 수정이나 마감처럼 자주 쓰는 기능을 사용자가 찾기 어려울 수 있어, 현재 단계에서는 추천도가 낮습니다.

추천 방향

A를 내가 만든 모임의 재진입 화면으로 두고, B와 C를 owner 관리/수정 화면으로 잡으며, D와 E를 공개 참석 화면의 입장 전/저장 후 상태로 적용하는 조합을 추천합니다. 모임 등록은 `/meetings/new` 한 화면 안의 2단계 form으로 만들고, 2단계의 `모임 만들기` 버튼에서 저장한 뒤 `/meetings/{publicId}/manage`로 이동하는 흐름이 명확합니다.

모임 등록 2단계 화면

`/meetings/new` 한 URL에서 1단계 기본 정보와 2단계 모임 설정을 모두 입력합니다. 단계는 URL query가 아니라 화면 내부 상태로만 전환되고, 생성이 완료되면 관리 화면으로 이동합니다.
